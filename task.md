# Техническое задание: Telegram-бот для организации медиафайлов через LLM

## 1. Цель

Создать простое приложение, которое принимает торрент-ссылки через Telegram-бота, скачивает файлы и автоматически организует их в медиатеку для Jellyfin.

LLM используется для принятия решений: определить тип контента, название, сезон, серию, качество, целевую папку и финальное имя файла.

---

## 2. Основной сценарий

1. Пользователь отправляет Telegram-боту magnet-ссылку или `.torrent` файл.
2. Бот добавляет торрент в загрузчик.
3. После завершения загрузки приложение анализирует скачанные файлы.
4. LLM определяет, что это:

   * фильм
   * сериал
   * сезон сериала
   * отдельная серия
   * лишние файлы
5. Приложение перемещает или копирует нужные видеофайлы в медиатеку.
6. Файлы переименовываются в формат, понятный Jellyfin.
7. Пользователь получает сообщение в Telegram с результатом.

---

## 3. Компоненты

### 3.1 Telegram Bot

Функции:

* принимать magnet-ссылки
* принимать `.torrent` файлы
* показывать статус загрузки
* уведомлять об успехе или ошибке
* принимать подтверждение, если LLM не уверен

Команды:

```text
/start
/status
/list
/help
```

---

### 3.2 Download Manager

Рекомендуемый вариант: qBittorrent Web API.

Функции:

* добавить торрент
* получить статус
* отследить завершение
* получить путь к скачанным файлам

---

### 3.3 Media Organizer

Функции:

* сканировать папку завершённой загрузки
* находить видеофайлы
* игнорировать мусор:

  * `.txt`
  * `.nfo`
  * `.jpg`
  * `.sample`
  * трейлеры
* вызывать LLM для классификации
* создавать структуру папок
* переносить файлы в медиатеку

## 4. Использование LLM

LLM получает список файлов и должен вернуть JSON.

Пример входа:

```json
{
  "torrent_name": "Breaking.Bad.S01.1080p.BluRay.x265",
  "files": [
    "Breaking.Bad.S01E01.1080p.BluRay.x265.mkv",
    "Breaking.Bad.S01E02.1080p.BluRay.x265.mkv",
    "sample.mkv",
    "readme.txt"
  ]
}
```

Пример ответа:

```json
{
  "content_type": "series",
  "title": "Breaking Bad",
  "year": 2008,
  "confidence": 0.95,
  "items": [
    {
      "source": "Breaking.Bad.S01E01.1080p.BluRay.x265.mkv",
      "target": "/media/series/Breaking Bad/Season 01/Breaking Bad - S01E01.mkv"
    },
    {
      "source": "Breaking.Bad.S01E02.1080p.BluRay.x265.mkv",
      "target": "/media/series/Breaking Bad/Season 01/Breaking Bad - S01E02.mkv"
    }
  ],
  "ignored_files": [
    "sample.mkv",
    "readme.txt"
  ]
}
```

## 5. Правила организации файлов

### Фильмы

```text
/media/movies/Movie Name (Year)/Movie Name (Year).mkv
```

Пример:

```text
/media/movies/Inception (2010)/Inception (2010).mkv
```

### Сериалы

```text
/media/series/Series Name/Season 01/Series Name - S01E01.mkv
```

Пример:

```text
/media/series/Breaking Bad/Season 01/Breaking Bad - S01E01.mkv
```

---

## 6. Безопасность

LLM не должен напрямую выполнять shell-команды.

Разрешено:

* вернуть JSON-план
* приложение валидирует JSON
* приложение само выполняет файловые операции

Запрещено:

* выполнять команды из ответа LLM
* писать за пределы разрешённых директорий
* перезаписывать существующие файлы без проверки

---

## 7. Подтверждение пользователем

Если `confidence < 0.8`, бот должен отправить пользователю план:

```text
Нашёл сериал:
Breaking Bad
Season 01
2 серии

Переместить в:
/media/series/Breaking Bad/Season 01/
```

Кнопки:

```text
✅ Подтвердить
❌ Отменить
✏️ Исправить вручную
```

## 10. MVP

В первую версию входит:

* Telegram-бот
* добавление magnet-ссылки
* интеграция с qBittorrent
* обработка завершённых загрузок
* LLM-классификация файлов
* перенос файлов в `/media/movies` или `/media/series`
* Telegram-уведомления
* ручное подтверждение при низкой уверенности

Не входит в MVP:

* поиск торрентов
* автоматический выбор качества
* подписки на сериалы
* web-интерфейс
* сложные правила апгрейда качества

---

## 11. Важное ограничение

LLM должен быть помощником для классификации и переименования, но не единственным источником истины. Все операции с файлами должны проходить через строгую валидацию приложения.
